CHAPTER 00
Why Transformers?
Before transformers, we had RNNs and LSTMs — they read words one by one, like reading a book with one eye closed. Transformers read EVERYTHING at once.
THE CORE PROBLEM
❌ OLD (RNN / LSTM)
1
Sequential
Reads word 1 → word 2 → word 3... slow, forgets early words
2
Vanishing Gradient
Long sentences = model forgets the beginning
3
Can't Parallelize
GPU can't speed it up — must wait for each word
✅ NEW (Transformer)
1
Parallel
ALL words processed at the same time — super fast on GPU
2
Full Context
Every word looks at every other word — no forgetting
3
Scales Massively
GPT, BERT, T5 — all built on this foundation
💡
Key Insight: "The cat sat on the mat because it was tired" — What is it? The cat or the mat?
Transformer looks at ALL words at once and figures out: it = cat (because "tired" connects to living things).

STEP 04
Learned Weight Matrices
This is where LEARNING happens. Three weight matrices (W_Q, W_K, W_V) are learned during training. They transform each token's embedding into Q, K, V vectors.
🧠
These are the LEARNED parameters. At the start of training, they are random. Over millions of training steps, backpropagation adjusts them until the model learns language. GPT-3 has 175 BILLION such numbers.

STEP 05
Attention Scores (Q × Kᵀ)
Now every token asks: "How much should I pay attention to every other token?" We compute this using dot products between Q and K vectors.
STEP 5A — Transpose of K (Kᵀ)
K (8×3) becomes Kᵀ (3×8) — rows become columns:

Kᵀ =
  [2 1 1 1 0 2 1 2] <— K dim 0
  [1 2 1 1 1 2 0 1] <— K dim 1
  [1 1 2 0 1 2 1 1] <— K dim 2
Shape: (3×8)
STEP 5B — Q × Kᵀ = (8×3)×(3×8) = (8×8) ATTENTION SCORES
score(i,j) = Q[i] · K[j]   (dot product)
Worked example — Row "I", Column "it":

score = 2×2 + 1×2 + 1×2 = 4+2+2 = 10

       ≈ [3.46, 2.89, 2.89, 1.73, 1.15, 5.77, 1.73, 3.46]
SOFTMAX: convert each row into probabilities
softmax(xi) = exi / Σ exj

STEP 08
Multi-Head Attention
Instead of ONE Q·K·V, we run MULTIPLE attention heads in parallel. Each head learns different relationships — grammar, meaning, co-reference, long-range dependencies.
WHY MULTIPLE HEADS?
HEAD 1
Grammar
subject → verb relationships
"I" → "love" → "machine"
HEAD 2
Co-reference
pronoun → noun links
"it" → "machine learning"
HEAD 3
Semantics
word meaning clusters
"magic" ↔ "love" (positive)
HEAD 4
Long-Range
distant word relations
"I" ↔ "magic" (ends)
MultiHead(Q,K,V) = Concat(head₁, head₂, head₃, ...) · W_O
⚡
In GPT-3: 96 attention heads, each with d_k=128. All run in PARALLEL on GPU.
Final output = Concatenate all heads → multiply by output weight matrix W_O → one combined vector per token.
⭐ INTERMEDIATE LEVEL
STEP 09
Training: How Weights Update
During training, the model makes predictions, calculates how wrong it was (loss), then uses backpropagation to adjust ALL weight matrices to reduce the error.
TRAINING LOOP
1
Forward Pass
Input "I love machine learning" → transformer predicts next word → "because" (probability distribution over vocab)
2
Calculate Loss
Correct answer: "because"   Predicted: "which"  → Cross-entropy loss = how wrong we were
Loss = -log(P(correct token))
3
Backpropagation
Gradients flow BACKWARDS through all layers — computing ∂Loss/∂W for every single weight
4
Weight Update (Gradient Descent)
W = W − η · ∇L
η = learning rate (e.g. 0.001)  |  ∇L = gradient
This slightly adjusts W_Q, W_K, W_V, FFN weights toward better predictions.

⭐⭐ ADVANCED LEVEL
STEP 10
Decoder — Generating Output
The encoder understood the input. Now the Decoder generates output ONE TOKEN AT A TIME — using three sublayers: masked self-attention (can't peek at future), cross-attention to the encoder, and a feed-forward network.
THE DECODER'S THREE SUBLAYERS
SUBLAYER 1
🔒 Masked Self-Attention
Each decoded token attends to ALL previously decoded tokens, but CANNOT see future tokens. Uses a causal mask (lower-triangular matrix).
SUBLAYER 2
🔗 Cross-Attention
Decoder Q comes from sublayer 1. Encoder K, V come from the encoder output. This is how decoder "reads" the source sentence.
SUBLAYER 3
⚡ Feed-Forward + Output
Same FFN as encoder. Then a final linear layer maps to vocab_size logits → softmax → next token probability.
🏗
Encoder vs Decoder: Encoder processes ALL input tokens simultaneously (bidirectional attention). Decoder generates tokens sequentially — one at a time — and can only attend to the past (causal/masked attention). Each sublayer is wrapped in Add & LayerNorm (residual connection).
🔒 WHAT IS THE MASK? — The Causal Mask Explained
During training the decoder receives the FULL target sequence at once (for efficiency).
But it must NOT cheat — token at position i cannot see tokens at position i+1, i+2, …
Solution: we ADD a large negative number (−∞) to the attention scores for all future positions before softmax.
① RAW SCORES (Q·Kᵀ/√dk) before mask — 4 tokens "I love ML magic"
② ADD MASK (future positions → −∞)
③ AFTER SOFTMAX — future positions become 0.00
💡
Why −∞? softmax(−∞) = e−∞ / Σ = 0 / Σ ≈ 0. Those positions become exactly zero probability — as if they don't exist. The model physically cannot attend to them. In practice, a very large negative number like −1e9 or −10000 is used.
WORKED EXAMPLE — Masked Attention for "love" (position 1)
Decoder sequence: "I love ML magic"  (positions 0, 1, 2, 3)
We compute attention for "love" at position 1. It can see: "I" ✅ and itself ✅. It CANNOT see: "ML" ✗ and "magic" ✗

ENCODER-DECODER CROSS-ATTENTION — How the decoder "reads" the source
For each generated token, cross-attention lets the decoder look at the encoder output:
· Q = from decoder (what concept are we generating now?)
· K, V = from encoder (what did the source sentence contain?)
Thicker lines = stronger attention. Each output word attends most to its corresponding source token.
⚡
Cross-attention is the bridge. Masked self-attention only gives context of decoded tokens so far. Cross-attention provides the source meaning. Without it, the decoder would be generating blindly — like translating without reading the original!

CHAPTER 12
Build & Use Transformers in TensorFlow
Two paths: build a custom Transformer from scratch using Keras layers, or load a pre-trained model from HuggingFace and run inference in seconds. Every block is explained — no magic.
BLOCK 1 — Scaled Dot-Product Attention
The core attention formula: Attention(Q,K,V) = softmax(QKᵀ / √dk) · V
Takes Query, Key, Value matrices → returns a context-weighted output matrix. 
# ── Block 1: Scaled Dot-Product Attention ──────────────────────
import tensorflow as tf
import numpy as np

def scaled_dot_product_attention(Q, K, V, mask=None):
    """
    Q : (batch, heads, seq_q, d_k)
    K : (batch, heads, seq_k, d_k)
    V : (batch, heads, seq_k, d_v)
    Returns output (batch, heads, seq_q, d_v) and weights
    """
    d_k = tf.cast(tf.shape(K)[-1], tf.float32)

    # Step 1 — Q · Kᵀ  →  (batch, heads, seq_q, seq_k)
    scores = tf.matmul(Q, K, transpose_b=True)   # raw dot products
    scores = scores / tf.math.sqrt(d_k)           # ÷ √dk  (stabilise gradients)

    # Step 2 — apply causal/padding mask (add -1e9 to blocked positions)
    if mask is not None:
        scores += (mask * -1e9)

    # Step 3 — softmax over last axis → attention weights (sum to 1 per row)
    weights = tf.nn.softmax(scores, axis=-1)     # (batch, heads, seq_q, seq_k)

    # Step 4 — weighted sum of Values
    output = tf.matmul(weights, V)                # (batch, heads, seq_q, d_v)
    return output, weights
💡
mask * -1e9 — where mask=1 (blocked), scores become ≈−∞, so softmax gives ≈0. This is exactly the causal mask from Step 10!
BLOCK 2 — Multi-Head Attention Layer
Splits input into num_heads parallel attention heads, runs scaled dot-product attention on each, then concatenates and projects back. Uses tf.keras.layers.Layer subclassing. 
# ── Block 2: Multi-Head Attention ───────────────────────────────
class MultiHeadAttention(tf.keras.layers.Layer):
    def __init__(self, d_model, num_heads):
        super().__init__()
        self.num_heads = num_heads
        self.d_model   = d_model
        assert d_model % num_heads == 0, "d_model must be divisible by num_heads"
        self.d_k = d_model // num_heads          # dimension per head

        # Learned projection matrices  W_Q, W_K, W_V, W_O
        self.W_Q = tf.keras.layers.Dense(d_model) # projects to all heads at once
        self.W_K = tf.keras.layers.Dense(d_model)
        self.W_V = tf.keras.layers.Dense(d_model)
        self.W_O = tf.keras.layers.Dense(d_model) # output projection

    def split_heads(self, x, batch_size):
        # x: (batch, seq, d_model) → (batch, heads, seq, d_k)
        x = tf.reshape(x, (batch_size, -1, self.num_heads, self.d_k))
        return tf.transpose(x, perm=[0, 2, 1, 3])

    def call(self, Q, K, V, mask=None):
        batch = tf.shape(Q)[0]

        Q = self.split_heads(self.W_Q(Q), batch)  # project + split
        K = self.split_heads(self.W_K(K), batch)
        V = self.split_heads(self.W_V(V), batch)

        # Attention for every head in parallel
        attn_out, attn_weights = scaled_dot_product_attention(Q, K, V, mask)

        # Merge heads back: (batch, heads, seq, d_k) → (batch, seq, d_model)
        attn_out = tf.transpose(attn_out, perm=[0, 2, 1, 3])
        attn_out = tf.reshape(attn_out, (batch, -1, self.d_model))

        output = self.W_O(attn_out)               # final linear projection
        return output, attn_weights
BLOCK 3 — Feed-Forward Network + Add & LayerNorm
Each sublayer output passes through Add & Norm: LayerNorm(x + sublayer(x)). The FFN is two linear layers with a GELU activation — it adds non-linearity and capacity. 
# ── Block 3: Feed-Forward Network ───────────────────────────────
def point_wise_ffn(d_model, dff):
    """dff = inner dimension (typically 4× d_model)"""
    return tf.keras.Sequential([
        tf.keras.layers.Dense(dff,     activation='gelu'),  # W1: d_model→dff
        tf.keras.layers.Dense(d_model, activation=None),   # W2: dff→d_model
    ])

# ── Block 3b: Encoder Layer (Attention + FFN + 2× Add&Norm) ─────
class EncoderLayer(tf.keras.layers.Layer):
    def __init__(self, d_model, num_heads, dff, dropout=0.1):
        super().__init__()
        self.mha  = MultiHeadAttention(d_model, num_heads)
        self.ffn  = point_wise_ffn(d_model, dff)
        self.ln1  = tf.keras.layers.LayerNormalization(epsilon=1e-6)
        self.ln2  = tf.keras.layers.LayerNormalization(epsilon=1e-6)
        self.drop1 = tf.keras.layers.Dropout(dropout)
        self.drop2 = tf.keras.layers.Dropout(dropout)

    def call(self, x, training, mask=None):
        # Sub-layer 1: Multi-Head Self-Attention + Add & Norm
        attn_out, _ = self.mha(x, x, x, mask)     # Q=K=V=x  (self-attention)
        attn_out    = self.drop1(attn_out, training=training)
        out1        = self.ln1(x + attn_out)       # residual connection

        # Sub-layer 2: FFN + Add & Norm
        ffn_out  = self.ffn(out1)
        ffn_out  = self.drop2(ffn_out, training=training)
        out2     = self.ln2(out1 + ffn_out)        # residual connection
        return out2
BLOCK 4 — Decoder Layer (3 sublayers)
The decoder has three sublayers: (1) masked self-attention, (2) cross-attention to encoder, (3) FFN. Each wrapped in Add & Norm. The causal mask prevents peeking at future tokens. 
# ── Block 4: Decoder Layer ──────────────────────────────────────
class DecoderLayer(tf.keras.layers.Layer):
    def __init__(self, d_model, num_heads, dff, dropout=0.1):
        super().__init__()
        self.mha1  = MultiHeadAttention(d_model, num_heads) # masked self-attn
        self.mha2  = MultiHeadAttention(d_model, num_heads) # cross-attention
        self.ffn   = point_wise_ffn(d_model, dff)
        self.ln1   = tf.keras.layers.LayerNormalization(epsilon=1e-6)
        self.ln2   = tf.keras.layers.LayerNormalization(epsilon=1e-6)
        self.ln3   = tf.keras.layers.LayerNormalization(epsilon=1e-6)
        self.drop1 = tf.keras.layers.Dropout(dropout)
        self.drop2 = tf.keras.layers.Dropout(dropout)
        self.drop3 = tf.keras.layers.Dropout(dropout)

    def call(self, x, enc_output, training, look_ahead_mask, padding_mask):
        # Sub-layer 1: Masked self-attention (uses look_ahead_mask)
        attn1, w1 = self.mha1(x, x, x, look_ahead_mask)
        attn1     = self.drop1(attn1, training=training)
        out1      = self.ln1(x + attn1)

        # Sub-layer 2: Cross-attention  Q←decoder, K,V←encoder
        attn2, w2 = self.mha2(out1, enc_output, enc_output, padding_mask)
        attn2     = self.drop2(attn2, training=training)
        out2      = self.ln2(out1 + attn2)

        # Sub-layer 3: Feed-Forward Network
        ffn_out   = self.ffn(out2)
        ffn_out   = self.drop3(ffn_out, training=training)
        out3      = self.ln3(out2 + ffn_out)
        return out3, w1, w2             # also return attn weights for viz
BLOCK 5 — Positional Encoding + Full Transformer Model
Sinusoidal positional encodings are added to the token embeddings before the first layer. Then we stack N encoder layers and N decoder layers to form the complete model. 
# ── Block 5a: Positional Encoding ───────────────────────────────
def positional_encoding(max_len, d_model):
    positions = np.arange(max_len)[:, np.newaxis]      # (max_len, 1)
    dims      = np.arange(d_model)[np.newaxis, :]      # (1, d_model)
    angles    = positions / np.power(10000, (2 * (dims // 2)) / d_model)
    angles[:, 0::2] = np.sin(angles[:, 0::2])  # even dims → sin
    angles[:, 1::2] = np.cos(angles[:, 1::2])  # odd  dims → cos
    return tf.cast(angles[np.newaxis, :, :], dtype=tf.float32)  # (1, max_len, d_model)

# ── Block 5b: Full Transformer ───────────────────────────────────
class Transformer(tf.keras.Model):
    def __init__(self, num_layers, d_model, num_heads, dff,
                 input_vocab, target_vocab, max_seq_len, dropout=0.1):
        super().__init__()
        self.enc_embed = tf.keras.layers.Embedding(input_vocab,  d_model)
        self.dec_embed = tf.keras.layers.Embedding(target_vocab, d_model)
        self.pos_enc   = positional_encoding(max_seq_len, d_model)
        self.enc_layers = [EncoderLayer(d_model, num_heads, dff, dropout)
                           for _ in range(num_layers)]
        self.dec_layers = [DecoderLayer(d_model, num_heads, dff, dropout)
                           for _ in range(num_layers)]
        self.final_layer = tf.keras.layers.Dense(target_vocab) # logits
        self.dropout     = tf.keras.layers.Dropout(dropout)

    def call(self, inputs, targets, training, enc_mask, look_ahead_mask, dec_mask):
        seq_len = tf.shape(inputs)[1]

        # ── Encoder ──────────────────────────────────────────
        x = self.enc_embed(inputs)                        # token → embedding
        x *= tf.math.sqrt(tf.cast(x.shape[-1], tf.float32)) # scale
        x += self.pos_enc[:, :seq_len, :]                 # add positional encoding
        x  = self.dropout(x, training=training)
        for layer in self.enc_layers:
            x = layer(x, training, enc_mask)

        enc_output = x  # encoder output → fed into every decoder cross-attn

        # ── Decoder ──────────────────────────────────────────
        tgt_len = tf.shape(targets)[1]
        y = self.dec_embed(targets)
        y *= tf.math.sqrt(tf.cast(y.shape[-1], tf.float32))
        y += self.pos_enc[:, :tgt_len, :]
        y  = self.dropout(y, training=training)
        for layer in self.dec_layers:
            y, _, _ = layer(y, enc_output, training, look_ahead_mask, dec_mask)

        logits = self.final_layer(y)  # (batch, tgt_seq, target_vocab)
        return logits

# ── Block 5c: Instantiate & compile ─────────────────────────────
transformer = Transformer(
    num_layers=4,           # N encoder + N decoder layers
    d_model=128,            # embedding / attention dimension
    num_heads=8,            # number of parallel attention heads
    dff=512,               # FFN inner dimension (4× d_model)
    input_vocab=8500,       # source vocabulary size
    target_vocab=8000,      # target vocabulary size
    max_seq_len=128,        # maximum sequence length
    dropout=0.1
)

transformer.compile(
    optimizer=tf.keras.optimizers.Adam(learning_rate=1e-4, beta_1=0.9, beta_2=0.98),
    loss=tf.keras.losses.SparseCategoricalCrossentropy(from_logits=True),
    metrics=['accuracy']
)
🏗
Model size: num_layers=4, d_model=128, num_heads=8, dff=512 gives ~3M parameters — trainable on a laptop GPU. GPT-2 small uses d_model=768, 12 heads, 12 layers (~117M params).
BLOCK 6 — Create Masks + Training Loop
# ── Block 6a: Mask helpers ──────────────────────────────────────
def create_padding_mask(seq):
    """Mask positions where token == 0 (padding)"""
    mask = tf.cast(tf.math.equal(seq, 0), tf.float32)
    return mask[:, tf.newaxis, tf.newaxis, :]  # (batch, 1, 1, seq)

def create_look_ahead_mask(size):
    """Lower-triangular causal mask — 1 = blocked (future), 0 = visible (past)"""
    mask = 1 - tf.linalg.band_part(tf.ones((size, size)), -1, 0)
    return mask  # upper-triangular 1s → these positions get -1e9

# ── Block 6b: Training step ─────────────────────────────────────
@tf.function
def train_step(inp, tar):
    # Teacher forcing: feed correct tokens as decoder input, shifted right
    tar_inp  = tar[:, :-1]   # decoder input  (all but last)
    tar_real = tar[:, 1:]   # target labels  (all but first)

    enc_mask       = create_padding_mask(inp)
    look_ahead     = create_look_ahead_mask(tf.shape(tar_inp)[1])
    dec_mask       = create_padding_mask(inp)
    combined_mask  = tf.maximum(look_ahead, create_padding_mask(tar_inp))

    with tf.GradientTape() as tape:
        logits = transformer(inp, tar_inp, True, enc_mask, combined_mask, dec_mask)
        loss   = tf.keras.losses.sparse_categorical_crossentropy(
                     tar_real, logits, from_logits=True)
        loss   = tf.reduce_mean(loss)

    grads = tape.gradient(loss, transformer.trainable_variables)
    transformer.optimizer.apply_gradients(
        zip(grads, transformer.trainable_variables))
    return loss

# ── Block 6c: Epoch loop ────────────────────────────────────────
for epoch in range(20):
    for (batch, (inp, tar)) in enumerate(train_dataset):
        loss = train_step(inp, tar)
    print(f"Epoch {epoch+1}  loss={loss:.4f}")
BLOCK 7 — Autoregressive Inference (greedy decode)
At inference there is no teacher forcing. We feed <START> and let the model generate one token at a time until it produces <END> or hits max_length. 
# ── Block 7: Greedy inference ───────────────────────────────────
def predict(model, inp_sentence, tokenizer_src, tokenizer_tgt, max_len=50):
    # 1. Tokenise source sentence
    inp_ids = tokenizer_src.encode(inp_sentence)
    inp_ids = tf.expand_dims(inp_ids, 0)                 # (1, src_len)

    # 2. Start with the <START> token id
    START_ID = tokenizer_tgt.vocab_size
    END_ID   = tokenizer_tgt.vocab_size + 1
    out      = tf.expand_dims([START_ID], 0)              # (1, 1)

    # 3. Autoregressive loop — generate one token at a time
    for _ in range(max_len):
        enc_mask      = create_padding_mask(inp_ids)
        look_ahead    = create_look_ahead_mask(tf.shape(out)[1])
        dec_mask      = create_padding_mask(inp_ids)
        combined_mask = tf.maximum(look_ahead, create_padding_mask(out))

        logits = model(inp_ids, out, False, enc_mask, combined_mask, dec_mask)
        # logits: (1, current_len, vocab_size) → take last position
        next_token = tf.argmax(logits[:, -1:, :], axis=-1)   # greedy
        out        = tf.concat([out, next_token], axis=1)

        if next_token == END_ID:
            break

    # 4. Decode token IDs back to text (strip START/END)
    predicted_ids = tf.squeeze(out, axis=0)[1:].numpy()
    return tokenizer_tgt.decode([i for i in predicted_ids if i < tokenizer_tgt.vocab_size])

# Example usage:
result = predict(transformer, "I love machine learning", tok_en, tok_de)
print(result)  # → "Ich liebe maschinelles Lernen"

USE CASE 1 — GPT-2 Text Generation (decoder-only)
GPT-2 is a decoder-only transformer. We load the pre-trained weights, tokenise a prompt, and autoregressively generate tokens. No fine-tuning needed for text completion. 
# ── GPT-2 Text Generation ───────────────────────────────────────
from transformers import TFGPT2LMHeadModel, GPT2Tokenizer
import tensorflow as tf

# 1. Load tokeniser and model weights from HuggingFace Hub
tokenizer = GPT2Tokenizer.from_pretrained('gpt2')
model     = TFGPT2LMHeadModel.from_pretrained('gpt2')

# 2. Encode your prompt → token IDs
prompt   = "I love machine learning because"
input_ids = tokenizer.encode(prompt, return_tensors='tf')

# 3. Generate (beam search for better quality)
output_ids = model.generate(
    input_ids,
    max_new_tokens=50,           # how many new tokens to produce
    num_beams=5,               # beam search width (1 = greedy)
    no_repeat_ngram_size=2,    # prevent repetition
    early_stopping=True,
    pad_token_id=tokenizer.eos_token_id
)

# 4. Decode back to text
generated = tokenizer.decode(output_ids[0], skip_special_tokens=True)
print(generated)
# → "I love machine learning because it gives computers the ability to
#    learn from data without being explicitly programmed..."
🔢
What model.generate() does internally: runs the decoder loop from Block 7 for you — feeding <START>, sampling or beam-searching, appending each new token, stopping at <EOS>. It is the same autoregressive loop, just packaged.
USE CASE 2 — BERT Sentence Embeddings (encoder-only)
BERT is encoder-only — bidirectional attention, no causal mask. It is used to get rich contextual embeddings of text, useful for classification, search, and similarity tasks. 
# ── BERT Sentence Embeddings ────────────────────────────────────
from transformers import TFBertModel, BertTokenizer
import tensorflow as tf

tokenizer = BertTokenizer.from_pretrained('bert-base-uncased')
model     = TFBertModel.from_pretrained('bert-base-uncased')

sentences = [
    "I love machine learning because it is magic",
    "Deep learning is fascinating and powerful",
]

# Tokenise with padding/truncation → attention_mask handles padding
encoded = tokenizer(sentences, padding=True, truncation=True,
                    max_length=64, return_tensors='tf')

# Forward pass — no gradients needed for inference
outputs = model(**encoded)

# outputs.last_hidden_state : (batch, seq_len, 768) — one vector per token
# outputs.pooler_output     : (batch, 768)          — [CLS] token embedding
token_embeddings   = outputs.last_hidden_state   # per-token context vectors
sentence_embedding = outputs.pooler_output       # whole-sentence vector

# Mean-pool token embeddings (often better than [CLS] for similarity)
mask         = tf.cast(encoded['attention_mask'][..., tf.newaxis], tf.float32)
mean_pooled  = tf.reduce_sum(token_embeddings * mask, axis=1) / tf.reduce_sum(mask, axis=1)

# Cosine similarity between two sentences
a, b = mean_pooled[0], mean_pooled[1]
cos_sim = tf.reduce_sum(a*b) / (tf.norm(a) * tf.norm(b))
print(f"Similarity: {cos_sim:.4f}")  # → ~0.91 (very similar topics)
USE CASE 3 — Fine-Tune BERT for Text Classification
Add a classification head on top of BERT's [CLS] token output and fine-tune on your labelled data. This is the standard approach for sentiment analysis, spam detection, etc. 
# ── Fine-Tune BERT for Classification ───────────────────────────
from transformers import TFBertForSequenceClassification, BertTokenizer
import tensorflow as tf

NUM_CLASSES = 2  # e.g. positive / negative sentiment

# 1. Load pre-trained BERT + classification head (randomly initialised)
tokenizer = BertTokenizer.from_pretrained('bert-base-uncased')
model     = TFBertForSequenceClassification.from_pretrained(
                'bert-base-uncased', num_labels=NUM_CLASSES)

# 2. Compile with a small learning rate (pre-trained weights are fragile)
model.compile(
    optimizer=tf.keras.optimizers.Adam(learning_rate=2e-5),
    loss=tf.keras.losses.SparseCategoricalCrossentropy(from_logits=True),
    metrics=['accuracy']
)

# 3. Tokenise your dataset
texts  = ["I love this film", "This was terrible", "Absolutely brilliant"]
labels = [1, 0, 1]  # 1=positive, 0=negative

encoded = tokenizer(texts, padding=True, truncation=True,
                    max_length=128, return_tensors='tf')
dataset = tf.data.Dataset.from_tensor_slices((dict(encoded), labels)).batch(8)

# 4. Fine-tune — just a few epochs needed on small data
model.fit(dataset, epochs=3)

# 5. Inference on new text
test = tokenizer(["This is pure magic"], return_tensors='tf', padding=True)
logits     = model(**test).logits           # (1, 2)
prediction = tf.argmax(logits, axis=-1)    # 0=negative, 1=positive
label      = ["negative", "positive"][prediction.numpy()[0]]
print(f"Predicted: {label}")  # → Predicted: positive
⚡
Learning rate matters! 2e-5 to 5e-5 is the standard range for BERT fine-tuning. Too high → catastrophic forgetting of pre-trained weights. Too low → won't adapt to your task.

T5 uses the full encoder-decoder architecture (like our custom transformer above). It frames every task as text-to-text using task prefixes. Perfect for summarisation, translation, Q&A.

        "being explicitly programmed. It focuses on developing computer programs "
        "that can access data and use it to learn for themselves.")
