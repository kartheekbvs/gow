CHAPTER 00
Why Activation Functions?
Without activation functions, a neural network — no matter how deep — is just a linear transformation. One matrix multiplication. Activation functions introduce non-linearity, letting networks learn curves, patterns, and complex decisions.
THE CORE PROBLEM — LINEAR NETWORKS ARE USELESS
❌ WITHOUT ACTIVATION (linear only)
1
Layer 1
output = W₁ · x + b₁
2
Layer 2
output = W₂·(W₁·x + b₁) + b₂
= (W₂W₁)·x + (W₂b₁+b₂)
= W·x + b — still just linear!
N
Any depth
Still equivalent to a SINGLE linear layer. Cannot model XOR, curves, images, language.
✅ WITH ACTIVATION FUNCTION
1
Non-linear transform
σ(W₁·x + b₁) bends the space
2
Compose non-linearities
Each layer warps the manifold differently → model can approximate ANY function (Universal Approximation Theorem)
∞
Expressive power
XOR ✅   Image classification ✅   Language ✅
💡
Universal Approximation Theorem: A neural network with at least one hidden layer and a non-linear activation can approximate any continuous function to arbitrary precision — given enough neurons. The activation function IS the source of this power.

FUNCTION 01
Sigmoid (Logistic)
The classic. Squashes any real number into (0, 1). Was the default activation for decades — until vanishing gradients killed it for deep networks. Still essential for binary output layers.
FORMULA & DERIVATIVE
Function
σ(x) = 1 / (1 + e−x)
Derivative (chain rule magic)
σ′(x) = σ(x) · (1 − σ(x))
Proof of derivative:
σ′(x) = d/dx (1+e−x)−1
= −(1+e−x)−2 · (−e−x)
= e−x / (1+e−x)²
= [1/(1+e−x)] · [e−x/(1+e−x)]
= σ(x) · (1 − σ(x)) ✓
Range
Output ∈ (0, 1) — never exactly 0 or 1
non-linear smooth probabilistic output saturates at ±∞ not zero-centred vanishing gradient

FUNCTION 02
Tanh (Hyperbolic Tangent)
A scaled, shifted sigmoid that outputs in (−1, 1). Zero-centred — gradients push both positive and negative. Preferred over sigmoid for hidden layers, but still saturates.
FORMULA & DERIVATIVE
Function
tanh(x) = (ex − e−x) / (ex + e−x)
Relation to Sigmoid
tanh(x) = 2σ(2x) − 1
Derivative
tanh′(x) = 1 − tanh²(x)
Proof of derivative:
tanh(x) = sinh(x)/cosh(x)
By quotient rule:
tanh′ = (cosh²x − sinh²x) / cosh²x
= 1 − sinh²x/cosh²x = 1 − tanh²(x) ✓

Max derivative = 1.0 at x=0 (vs 0.25 for sigmoid!)
zero-centred ✓ stronger gradient smooth still saturates exp() is slow

Tanh vs Sigmoid: Tanh output is zero-centred, so positive and negative updates cancel less during training. Max gradient is 4× larger (1.0 vs 0.25). Both still saturate for large |x|, both use expensive exp(). In practice: tanh for RNNs/LSTMs, neither for deep feedforward nets.
FUNCTION 03
ReLU (Rectified Linear Unit)
The most popular activation. Trivially simple — zero for negatives, identity for positives. Yet it unlocked training very deep networks. One comparison operation. No exp(). No saturation for positive inputs.
FORMULA & DERIVATIVE
Function
ReLU(x) = max(0, x)
Piecewise form
ReLU(x) = x if x > 0, else 0
Derivative
ReLU′(x) = 1 if x > 0, else 0
(undefined at x=0, treated as 0 or 1)
Why it works:
No saturation for x>0 → gradient always 1
Sparse activation (~50% neurons off)
Cheap: just max(0,x)
GPU-friendly: no exp() calls

Dead ReLU problem: if a neuron's input is always negative, its gradient is always 0 — it never updates. "Dying neurons."
no vanishing gradient (x>0) computationally cheapest sparse activation dead neurons not zero-centred not differentiable at 0

Sparsity is a feature: ~50% of ReLU neurons output 0 at any given pass. This creates sparse representations — fewer active neurons means less interference between features. Sparse codes are efficient and often more interpretable. This is one reason ReLU outperforms tanh/sigmoid on deep networks.
FUNCTION 04
Leaky ReLU & Parametric ReLU (PReLU)
A simple fix for dying ReLU neurons — instead of zero for negatives, use a small slope α. Neurons can never truly "die." PReLU makes α a learned parameter.
FORMULA & DERIVATIVE
Leaky ReLU (α typically 0.01)
f(x) = x if x>0, else αx
Compact form
f(x) = max(αx, x)
Derivative
f′(x) = 1 if x>0, else α
With α=0.01:
· Positive side: same as ReLU, gradient=1
· Negative side: small leak, gradient=0.01
· No dead neurons — gradient never exactly 0

PReLU: α is learned via backprop.
Each neuron learns its own optimal slope.
Adds very few extra parameters.
no dead neurons negative gradient ≠ 0 almost as cheap as ReLU α is a hyperparameter not zero-centred

FUNCTION 05
ELU (Exponential Linear Unit)
Smoother than ReLU for negatives. Negative outputs push mean activations toward zero, speeding up learning. Uses exp() for negatives — computationally heavier than ReLU.
FORMULA & DERIVATIVE
Function (α typically 1.0)
ELU(x) = x if x≥0
         α(ex−1) if x<0
Derivative
ELU′(x) = 1 if x≥0
          ELU(x) + α if x<0
Why α(eˣ−1)?
· Continuous and smooth at x=0
· Negative mean output → zero-centred
· As x→−∞, ELU→−α (soft saturation)
· Derivative is continuous at x=0: ELU′(0⁻)=α(e⁰−1)+α=0+α=α... wait:
  ELU′(0⁺)=1, ELU′(0⁻)=e⁰·α=α. If α=1 → smooth!

SELU (Scaled ELU): specific α,λ values that guarantee self-normalising activations (zero mean, unit variance).
smooth at x=0 negative outputs (zero-centred) no dead neurons exp() cost for negatives soft saturation for x≪0

⭐ MODERN — used in GPT, BERT, Transformers
FUNCTION 06
GELU (Gaussian Error Linear Unit)
The activation function of modern transformers (BERT, GPT-2/3/4, ViT). Rather than a hard gate (ReLU) it applies a smooth stochastic gate — inputs are weighted by the probability that a Gaussian input would be positive at that value.
FORMULA & DERIVATIVE
Exact formula
GELU(x) = x · Φ(x)
where Φ(x) = P(X≤x), X~N(0,1)
Fast approximation (used in practice)
GELU(x) ≈ 0.5x(1 + tanh(√(2/π)·(x+0.044715x³)))
Derivative (exact)
GELU′(x) = Φ(x) + x·φ(x)
where φ(x) = (1/√2π)·e−x²/2 (Gaussian PDF)
Intuition: GELU(x) = x × P(a random Gaussian ≤ x)
· Large positive x → P≈1 → passes through (like ReLU)
· Large negative x → P≈0 → suppressed
· Near 0: smooth probabilistic weighting
· Can be slightly negative (unlike ReLU)

Why better than ReLU? Smooth everywhere, non-monotonic near 0. Empirically outperforms ReLU on transformers and large models.
smooth everywhere non-monotonic probabilistic interpretation state-of-the-art for NLP more expensive than ReLU

⭐ MODERN — discovered by Neural Architecture Search
FUNCTION 07
Swish (SiLU)
Discovered by Google Brain via automated neural architecture search in 2017. Self-gated: the input gates itself through a sigmoid. Non-monotonic, smooth, unbounded above. Used in EfficientNet, PaLM, many modern vision models.
FORMULA & DERIVATIVE
Function (β=1 in practice)
Swish(x) = x · σ(βx) = x · (1/(1+e−βx))
Also known as SiLU (Sigmoid Linear Unit)
SiLU(x) = x · σ(x)
Derivative (product rule)
Swish′(x) = σ(x) + x·σ(x)·(1−σ(x))
           = σ(x) + Swish(x)·(1−σ(x))
Self-gating intuition:
σ(x) is the gate: near 0 for large negatives, near 1 for large positives.
x is the value that passes through the gate.
Result: large positives pass, near-zero inputs are smoothly gated, very negative values are suppressed (but non-zero)

Non-monotonic: has a small dip below zero near x≈−1.3, creating a "negative bump" that helps training.
smooth everywhere non-monotonic self-gated NAS-discovered sigmoid cost β is tunable

FUNCTION 08
Softmax
Not a neuron-wise activation — it operates on a whole vector at once. Turns a vector of arbitrary real numbers into a valid probability distribution. Always used as the final layer in multi-class classification and in attention mechanisms.
FORMULA & JACOBIAN
Function (applied to vector z ∈ ℝᴷ)
softmax(zᵢ) = ezᵢ / Σⱼ ezⱼ
Properties (always true)
Σᵢ softmax(zᵢ) = 1   (sums to 1)
softmax(zᵢ) ∈ (0, 1)   (all positive)
Jacobian (matrix of partial derivatives)
∂sᵢ/∂zⱼ = sᵢ(1−sᵢ) if i=j
          −sᵢ·sⱼ if i≠j
Temperature scaling:
softmax(z/T): T>1 → softer (uniform), T<1 → sharper (peaked)
Attention uses T=√d_k to stabilise

Numerical stability trick:
Always compute softmax(z − max(z))
This prevents overflow: every_large = inf
outputs sum to 1 probabilistic interpretation used in attention vector operation (not element-wise) can saturate (winner-takes-all)
GRAPH — Softmax over varying z₁, z₂=1, z₃=0
As z₁ increases it "steals" probability from the other classes
WORKED EXAMPLE — 3-class classification logits [2, 1, 0.5]
Logits z = [2.0, 1.0, 0.5]
Step 1 — exponentiate:
  e2.0 = 7.389,   e1.0 = 2.718,   e0.5 = 1.649
Step 2 — sum: 7.389 + 2.718 + 1.649 = 11.756
Step 3 — divide:
  P(class 1) = 7.389 / 11.756 = 0.6284
  P(class 2) = 2.718 / 11.756 = 0.2312
  P(class 3) = 1.649 / 11.756 = 0.1403
  Sum check: 0.6284 + 0.2312 + 0.1403 = 1.0000 ✓

Temperature scaling in Attention: Attention scores are divided by √d_k before softmax. With d_k=64, √d_k=8. This prevents very large dot products from pushing softmax into its saturated regions where gradients vanish — exactly the same problem as sigmoid!

CHAPTER 09
Which One to Use?
A practical decision guide — match the activation to your architecture, task, and problem.
DECISION GUIDE
1
Building a Transformer (BERT, GPT, ViT)?
Use GELU — it's the de-facto standard. TF: activation='gelu'. All major transformer implementations default to it. Swish is a good alternative.
2
Building a CNN (image classification, detection)?
Use ReLU as your default. If you see dying neurons (many weights stuck at zero), switch to Leaky ReLU (α=0.1) or Swish (used in EfficientNet).
3
Building an RNN or LSTM?
Use Tanh for the recurrent gate (preserves zero-centred activations through time). Use Sigmoid for gate controls (forget/input/output gates need 0–1 range).
4
Output layer — binary classification?
Use Sigmoid — outputs a single probability in (0,1). Threshold at 0.5 to get class prediction. Pair with BinaryCrossentropy(from_logits=False).
5
Output layer — multi-class classification?
Use Softmax — outputs a K-dimensional probability vector. Pair with SparseCategoricalCrossentropy(from_logits=False) in TensorFlow.
6
Output layer — regression (predict a number)?
Use no activation (linear). Let the output be unbounded. Use MeanSquaredError loss. Adding sigmoid/tanh here would clip your predictions to a fixed range.
7
Not sure? Baseline rule of thumb:
Start with ReLU → if it converges poorly, try GELU → if dying neurons, try Leaky ReLU. Run ablations for 5 epochs before committing.

# ── Activation functions in Keras ───────────────────────────────
import tensorflow as tf

# As layer argument (most common)
tf.keras.layers.Dense(128, activation='relu')       # ReLU
tf.keras.layers.Dense(128, activation='tanh')       # Tanh
tf.keras.layers.Dense(128, activation='sigmoid')    # Sigmoid
tf.keras.layers.Dense(128, activation='gelu')       # GELU ← transformers default
tf.keras.layers.Dense(128, activation='swish')      # Swish / SiLU
tf.keras.layers.Dense(128, activation='elu')        # ELU
tf.keras.layers.Dense(128, activation='softmax')    # Softmax (output layer)

# Leaky ReLU needs its own layer
tf.keras.layers.LeakyReLU(alpha=0.1)

# As standalone function (apply to tensors directly)
x = tf.constant([-2.0, -1.0, 0.0, 1.0, 2.0])
tf.nn.relu(x)           # → [0, 0, 0, 1, 2]
tf.nn.sigmoid(x)        # → [0.119, 0.269, 0.5, 0.731, 0.881]
tf.nn.tanh(x)           # → [-0.964, -0.762, 0, 0.762, 0.964]
tf.nn.gelu(x)           # → [-0.045, -0.159, 0, 0.841, 1.954]
tf.nn.swish(x)          # → [-0.238, -0.269, 0, 0.731, 1.762]
tf.nn.softmax(x)        # → probabilities summing to 1

# Numerical stability trick for softmax
x_stable = x - tf.reduce_max(x)  # subtract max before softmax
tf.nn.softmax(x_stable)           # same result, no overflow
f(x) = max(0, x)
