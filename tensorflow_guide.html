⬡ Chapter 01 — Foundations
CONCEPT 01
tf.Tensor — The Basics
A Tensor is the fundamental data structure in TensorFlow. Think of it as an n-dimensional array — a generalization of scalars, vectors, and matrices. Everything in TF flows through tensors.
What is a Tensor?
0D
Scalar
Single number. Shape = (). Example: tf.constant(42)
1D
Vector
Array of numbers. Shape = (n,). Example: tf.constant([1,2,3])
2D
Matrix
Rows × columns. Shape = (rows, cols). Like a spreadsheet of numbers.
nD
Higher-Order
Images: (batch, H, W, channels). Videos: (batch, T, H, W, C).
⚡ Live Tensor Inspector
💡
Immutability: Tensors created with tf.constant() are immutable — their values cannot change after creation. For learnable weights, use tf.Variable.

shapetuple/TensorShapeNone (inferred)Explicit reshape of value. Must be compatible with value's total elements.
namestrNoneOptional name for the tensor, used in TensorBoard and debugging.

import tensorflow as tf

# ── Creation ──────────────────────────────────
scalar = tf.constant(42)                          # 0D, shape=()

matrix = tf.constant([[1,2],[3,4]], dtype=tf.float32)  # 2D

# ── Inspect ───────────────────────────────────
print(matrix.shape)      # TensorShape([2, 2])
print(matrix.dtype)      # <dtype: 'float32'>
print(tf.rank(matrix))   # tf.Tensor(2, shape=(), dtype=int32)
print(tf.size(matrix))   # tf.Tensor(4, ...)

# ── Convert ───────────────────────────────────
np_arr = matrix.numpy()  # → numpy array [[1. 2.][3. 4.]]
f64 = tf.cast(matrix, tf.float64)

# ── Reshape ───────────────────────────────────
flat = tf.reshape(matrix, [-1])    # [1, 2, 3, 4]
col  = tf.expand_dims(vector, axis=1) # (3,) → (3,1)
CONCEPT 02
tf.Variable — Learnable Parameters
Variables are mutable tensors. They are the parameters your model learns: weights, biases, embeddings. When you call optimizer.apply_gradients(), it updates Variables in-place.
tf.Variable() — All Parameters
ParameterTypeDefaultDescription
initial_valuetensor/callablerequiredStarting value. Can be a tf.Tensor, numpy array, Python value, or a callable (lazy init).
trainableboolTrueIf True, GradientTape watches this variable and optimizers update it. Set False to freeze layers.
dtypetf.DTypeNoneData type. Usually float32 for weights.
namestr"Variable"Human-readable name shown in TensorBoard.
shapetupleNoneIf set, variable can be assigned values of any compatible shape.
Variable Methods
MethodDescription
.assign(value)Replace variable's value entirely. Returns the variable itself.
.assign_add(delta)Add delta to variable in-place: var += delta.
.assign_sub(delta)Subtract delta from variable: var -= delta.
.read_value()Returns current tensor value as a tf.Tensor snapshot.
.numpy()Get current value as numpy array.
.trainableBoolean attribute: is this variable watched by gradient tape?
⚠️
trainable=False — Freezing Layers: When you load a pretrained model and want to freeze its weights, set each layer's layer.trainable = False. This sets all Variable's trainable=False so they don't get updated during fine-tuning.
✅ tf.constant
  • Immutable — value never changes
  • Used for fixed inputs, hyperparams
  • Slightly faster (no mutation overhead)
  • Cannot be trained/updated by optimizer
  • Example: padding value, class weights
🔥 tf.Variable
  • Mutable — can be updated in-place
  • Used for weights, biases, embeddings
  • Watched by GradientTape automatically
  • Updated by optimizer.apply_gradients()
  • Example: Dense layer kernel, LSTM gates
# ── Create Variables ──────────────────────────
w = tf.Variable(tf.random.normal([3, 2]), name="weights")
b = tf.Variable(tf.zeros([2]), name="bias")

# ── Mutation ──────────────────────────────────
w.assign(tf.ones([3, 2]))         # replace all values
b.assign_add([0.1, 0.1])          # b += [0.1, 0.1]
b.assign_sub([0.05, 0.05])        # b -= [0.05, 0.05]

# ── Freeze (non-trainable) ────────────────────
frozen = tf.Variable([1.0, 2.0], trainable=False)
# Will NOT be updated by optimizer

CONCEPT 05 + 06
tf.function & GradientTape
tf.function compiles Python functions into TensorFlow graphs for speed. GradientTape records operations so you can compute gradients — the foundation of all training.
tf.function — Parameters
ParameterDefaultDescription
funcNoneThe Python function to compile. Used as decorator or directly.
input_signatureNoneList of tf.TensorSpec to fix input shapes/dtypes, preventing retracing on new shapes.
reduce_retracingFalseTry to reduce retracing by generalizing input specs automatically.
jit_compileFalseUse XLA compilation for extra speed (TPU/GPU). Can break on unsupported ops.
experimental_relax_shapesFalse(Legacy) generalize shapes to avoid retracing.
⚠️
Retracing: Every new input shape or Python value triggers retracing (slow!). Use input_signature or tf.TensorSpec to prevent this.
GradientTape — API
Parameter/MethodDescription
persistent=FalseIf True, tape can compute multiple gradients (e.g., for Jacobian). Must call tape.reset() when done.
watch_accessed_variables=TrueAutomatically watches all trainable Variables. Set False to manually control what's watched.
tape.watch(tensor)Manually watch a non-Variable tensor. Needed for input gradients.
tape.gradient(y, x)Compute dy/dx. x can be a tensor or list of tensors.
tape.jacobian(y, x)Full Jacobian matrix — all partial derivatives.
tape.batch_jacobian(y, x)Per-sample Jacobian for batched inputs (faster than looping).
# ── tf.function — compile for speed ──────────
@tf.function(input_signature=[tf.TensorSpec(shape=[None, 10], dtype=tf.float32)])
def fast_forward(x):
    return tf.matmul(x, weights) + bias

# ── GradientTape — compute gradients ─────────
x = tf.Variable([2.0, 3.0])

with tf.GradientTape() as tape:
    y = x[0]**2 + x[1]**3   # y = x0² + x1³

grads = tape.gradient(y, x)   # [2*x0, 3*x1²] = [4.0, 27.0]

# ── Persistent tape — multiple gradients ─────
with tf.GradientTape(persistent=True) as tape:
    y = x[0]**2
    z = x[1]**2

dy = tape.gradient(y, x)
dz = tape.gradient(z, x)   # Only possible with persistent=True
del tape                      # Always delete persistent tapes!

# ── Watch non-Variable (e.g., input gradients)
x_input = tf.constant([1.0, 2.0])
with tf.GradientTape() as tape:
    tape.watch(x_input)       # Must manually watch constants
    out = tf.reduce_sum(x_input ** 2)

grad = tape.gradient(out, x_input)   # [2.0, 4.0]

Keras Sequential API
The simplest way to build models — a linear stack of layers. Each layer feeds directly into the next. Perfect for most classification and regression tasks.

+ Softmax

Output

from tensorflow import keras

# ── Method 1: Pass layers as list ─────────────
model = keras.Sequential([
    keras.layers.Input(shape=(784,)),       # explicit input (recommended)
    keras.layers.Dense(256, activation="relu"),
    keras.layers.Dropout(0.3),
    keras.layers.Dense(128, activation="relu"),
    keras.layers.Dense(10, activation="softmax"),
], name="mnist_classifier")

# ── Method 2: .add() incrementally ────────────
model2 = keras.Sequential()
model2.add(keras.layers.Dense(64, activation="relu", input_shape=(20,)))
model2.add(keras.layers.Dense(1, activation="sigmoid"))

# ── Inspect ───────────────────────────────────
model.summary()                  # print architecture
model.layers                     # list of layer objects
model.get_layer("dense").weights  # access layer weights
CONCEPT 08
Keras Functional API
Graph-style model building. Create any DAG architecture: multi-input, multi-output, residual connections, skip connections. More flexible than Sequential — used for ResNets, BERT, etc.
When to use Functional API
  • ✅ Residual / skip connections (ResNet)
  • ✅ Multiple inputs (text + image)
  • ✅ Multiple outputs (classification + regression)
  • ✅ Shared layers (weight sharing)
  • ✅ When you need model.get_layer() access
  • ❌ Loops / conditionals inside forward pass → use Subclassing
# ── Multi-input Functional model ─────────────
img_in = keras.Input(shape=(224,224,3), name="image")
txt_in = keras.Input(shape=(100,), name="text")

# Image branch
x = keras.layers.Conv2D(32, 3, activation="relu")(img_in)
x = keras.layers.GlobalAveragePooling2D()(x)

# Text branch
t = keras.layers.Embedding(10000, 64)(txt_in)
t = keras.layers.GlobalAveragePooling1D()(t)

# Merge & output
merged = keras.layers.Concatenate()([x, t])
out = keras.layers.Dense(1, activation="sigmoid")(merged)

model = keras.Model(
    inputs=[img_in, txt_in],
    outputs=out
)
CONCEPT 09
Keras Subclassing API
Maximum flexibility. Define models as Python classes. The call() method IS the forward pass — you can use Python loops, conditionals, and any logic inside it.
🧠
Use Subclassing when: you need dynamic computation graphs — different paths based on input, RNNs with custom recurrence, research architectures with experimental ops.
class MyTransformerBlock(keras.layers.Layer):
    def __init__(self, d_model, num_heads, dff, dropout_rate=0.1):
        super().__init__()
        self.attn = keras.layers.MultiHeadAttention(num_heads=num_heads, key_dim=d_model//num_heads)
        self.ff1  = keras.layers.Dense(dff, activation="gelu")
        self.ff2  = keras.layers.Dense(d_model)
        self.norm1 = keras.layers.LayerNormalization()
        self.norm2 = keras.layers.LayerNormalization()
        self.drop  = keras.layers.Dropout(dropout_rate)

    def call(self, x, training=False):
        # Self-attention + residual
        attn_out = self.attn(x, x)
        x = self.norm1(x + self.drop(attn_out, training=training))
        # Feed-forward + residual
        ff_out = self.ff2(self.ff1(x))
        x = self.norm2(x + self.drop(ff_out, training=training))
        return x

    def get_config(self):       # For serialization!
        return {"d_model": ..., "num_heads": ...}

# ── build() vs __init__() ─────────────────────
class LinearLayer(keras.layers.Layer):
    def build(self, input_shape):    # called once on first data
        self.w = self.add_weight(shape=(input_shape[-1], self.units),
                                  initializer="glorot_uniform")
        self.b = self.add_weight(shape=(self.units,), initializer="zeros")

CONCEPT 10
Dense, Embedding, Attention Layers
Dense Layer — keras.layers.Dense()
Fully-connected layer. Every input neuron connects to every output neuron. Computes: output = activation(input @ kernel + bias)
ParameterTypeDefaultDescription
unitsintrequiredNumber of output neurons. The "width" of this layer.
activationstr / callableNone"relu", "sigmoid", "tanh", "softmax", "gelu". None = linear (no activation).
use_biasboolTrueWhether to add a bias vector. Set False if using BatchNorm after.
kernel_initializerstr / Initializer"glorot_uniform"How weights are initialized. Options: "glorot_normal", "he_normal", "zeros", "orthogonal".
bias_initializerstr / Initializer"zeros"How bias is initialized.
kernel_regularizerRegularizerNonekeras.regularizers.L2(0.01) to penalize large weights (prevents overfitting).
activity_regularizerRegularizerNoneApplied to the layer's output (activations), not weights.
kernel_constraintConstraintNonekeras.constraints.MaxNorm(2.) — clip weights during training.
🔥 Embedding Layer — keras.layers.Embedding()
Maps integer token IDs to dense vectors. Learns a lookup table: each word/token gets its own trainable vector. Input shape: (batch, seq_len). Output: (batch, seq_len, embedding_dim).
ParameterTypeDefaultDescription
input_dimintrequiredVocabulary size — number of unique tokens. All inputs must be < input_dim.
output_dimintrequiredEmbedding dimension (vector size). Common: 64, 128, 256, 512, 768 (BERT).
embeddings_initializerstr"uniform"How to initialize embedding vectors. Use "glorot_uniform" for better training.
mask_zeroboolFalseIf True, token ID 0 is treated as padding and masked from loss/attention.
input_lengthintNoneFixed sequence length (needed for Flatten after Embedding).
embeddings_regularizerRegularizerNoneL2 regularization on embedding weights.
trainableboolTrueSet False to use frozen pretrained embeddings (e.g., GloVe, word2vec).
🔥 MultiHeadAttention — keras.layers.MultiHeadAttention()
The core of Transformer models. Splits queries, keys, values into multiple "heads" that attend in parallel, then concatenates and projects the results. Learns relationships across positions.
ParameterTypeDefaultDescription
num_headsintrequiredNumber of parallel attention heads. Common: 4, 8, 12 (BERT-base=12, GPT-3=96).
key_dimintrequiredSize of each attention head's key/query. Often d_model // num_heads.
value_dimintNoneDimension for value projections. Defaults to key_dim if None.
dropoutfloat0.0Dropout applied to attention weights (not values). Typically 0.1 for training.
use_biasboolTrueWhether to use bias in Q,K,V projections.
output_shapeint/tupleNoneOverride output projection dimension. Defaults to input's last dim.
attention_axesint/tupleNoneWhich axes to attend over. None = attend over all but batch and head.
📐
Call signature: mha(query, value, key=None, attention_mask=None, training=False, return_attention_scores=False). If key=None, uses value as key (self-attention). attention_mask shape: (batch, num_heads, q_len, k_len).
🔥 LayerNormalization
ParameterDefaultDescription
axis-1Which axis (or list of axes) to normalize. -1 = last axis (per-token in transformers).
epsilon0.001Small constant added to variance to prevent division by zero.
centerTrueWhether to learn β (bias) shift parameter.
scaleTrueWhether to learn γ (scale) parameter.
Formula
y = γ × (x − μ) / √(σ² + ε) + β
μ,σ computed per token (across features)
Dropout
ParameterDefaultDescription
raterequiredFraction of inputs to randomly set to 0. 0.2–0.5 typical. 0 = no dropout.
noise_shapeNoneShape of binary dropout mask. For sequence data, use (batch,1,features) to drop whole timesteps.
seedNoneRandom seed for reproducibility.
trainingFalseIn call(): must pass training=True to activate dropout during training!
⚠️
Dropout is ONLY active during training — automatically disabled during model.evaluate() and model.predict().
CONCEPT 12
Activation Functions
Activation functions introduce non-linearity. Without them, deep networks collapse to a single linear transformation. Each function has different properties that suit different tasks.
⚡ Interactive Activation Comparison
ReLU & Variants
ReLU
f(x) = max(0, x)
Leaky ReLU
f(x) = x if x>0 else αx (α≈0.01–0.3)
GELU (Transformers)
f(x) = x·Φ(x) where Φ = CDF of normal
💡
ReLU dying neuron problem: If a neuron always receives negative inputs, its gradient is always 0 — it never learns. Fix: use Leaky ReLU, He initialization, or smaller learning rates.
Sigmoid & Tanh
Sigmoid
σ(x) = 1/(1+e⁻ˣ) → range [0,1]
Tanh
tanh(x) = (eˣ−e⁻ˣ)/(eˣ+e⁻ˣ) → range [−1,1]
⚠️
Vanishing gradient: For large |x|, gradients of sigmoid/tanh → 0. Avoid in hidden layers of deep networks — use ReLU variants instead. Sigmoid still used for binary output layers.
Softmax
Softmax
P(i) = eˣⁱ / Σⱼ eˣʲ
ParameterDescription
axisAxis to normalize along. Default -1 (last axis). In NLP use axis=-1 over vocab dimension.
✅
Temperature scaling: Divide logits by T before softmax. T>1 → uniform distribution, T<1 → sharper/confident predictions.
⬡ Chapter 04 — Loss Functions & Optimizers
CONCEPT 13
Loss Functions
Regression Losses
MSE — Mean Squared Error
L = (1/n)Σ(y − ŷ)²
Penalizes large errors heavily. Sensitive to outliers.
MAE — Mean Absolute Error
L = (1/n)Σ|y − ŷ|
More robust to outliers. Less smooth near 0.
Huber Loss
Quadratic for |err|≤δ, linear otherwise.
Best of both worlds. δ is a hyperparameter.
Classification Losses
Binary Crossentropy
L = −[y·log(ŷ) + (1−y)·log(1−ŷ)]
For 2-class problems. Output: sigmoid neuron.
Categorical Crossentropy
L = −Σᵢ yᵢ·log(ŷᵢ)
Multi-class, one-hot labels.
Sparse Cat. Crossentropy
Same as above but labels are integers.
More memory-efficient, no one-hot needed.
Loss Parameters
ParameterDescription
from_logitsIf True, applies softmax/sigmoid internally. More numerically stable. Use when last layer has NO activation.
reduction"sum_over_batch_size" (default), "sum", "none". Use "none" for per-sample losses in custom loops.
nameHuman-readable name for this loss in logs.
label_smoothingBCELoss: fraction to smooth labels. 0.1 means y=0→0.05, y=1→0.95. Reduces overconfidence.
class_weightPass to model.fit() as dict {0:1.0, 1:5.0}. Weight rare classes higher.
CONCEPT 14
Optimizers — Adam & Beyond
Optimizers update model weights based on gradients. Adam is the default workhorse. Each optimizer has specific hyperparameters that significantly affect training.
🔥 Adam — keras.optimizers.Adam()
Adaptive Moment Estimation. Combines momentum and RMSprop. Maintains per-parameter adaptive learning rates. Works well out of the box.
ParameterTypeDefaultDescription
learning_ratefloat / Schedule0.001Step size for weight updates. Most important hyperparameter. Try: 1e-4 to 1e-2.
beta_1float0.9Exponential decay for first moment (gradient momentum). Controls how quickly old gradients decay.
beta_2float0.999Exponential decay for second moment (squared gradient). Controls step size adaptation.
epsilonfloat1e-7Prevents division by zero. Increase (e.g., 1e-5) if training is unstable or using float16.
amsgradboolFalseUse AMSGrad variant — keeps running max of squared gradients for convergence guarantees.
weight_decayfloatNoneL2 weight decay (AdamW behavior when set). Decouples weight decay from gradient update.
clipnormfloatNoneClip gradients by global norm. Prevents exploding gradients in RNNs. Typical: 1.0.
clipvaluefloatNoneClip each gradient individually to [-clipvalue, clipvalue].
global_clipnormfloatNoneClip by global norm across all gradients simultaneously (stronger regularization).
SGD — Stochastic Gradient Descent
ParameterDefaultDescription
learning_rate0.01Step size. Usually higher than Adam (0.01–0.1).
momentum0.0Accumulates gradient direction. 0.9 typical. Helps escape local minima.
nesterovFalseNesterov Accelerated Gradient — "lookahead" momentum. Often better than standard momentum.
weight_decayNoneL2 weight decay (regularization).
Learning Rate Schedules
ScheduleDescription
ExponentialDecaylr × decay_rate^(step/decay_steps). Smoothly decreasing LR.

    optimizer=keras.optimizers.Adam(learning_rate=1e-3),
    loss=keras.losses.SparseCategoricalCrossentropy(from_logits=True),
    metrics=["accuracy", keras.metrics.TopKCategoricalAccuracy(k=5)]
)

# ── Multi-output compile ─────────────────────
model.compile(
    optimizer="adam",
    loss={"class_out": "categorical_crossentropy", "bbox_out": "mse"},
    loss_weights={"class_out": 1.0, "bbox_out": 5.0},
    metrics={"class_out": ["accuracy"], "bbox_out": ["mae"]}
)

# ── Debug mode ───────────────────────────────
model.compile(optimizer="adam", loss="mse", run_eagerly=True)

optimizer = keras.optimizers.Adam(1e-3)

@tf.function  # compile for speed!
def train_step(x_batch, y_batch):
    with tf.GradientTape() as tape:
        logits = model(x_batch, training=True)   # forward pass
        loss   = loss_fn(y_batch, logits)           # compute loss
        loss  += sum(model.losses)                 # add regularization losses!

    grads = tape.gradient(loss, model.trainable_variables)
    optimizer.apply_gradients(zip(grads, model.trainable_variables))
    train_acc.update_state(y_batch, logits)
    return loss

for epoch in range(50):

    for x_batch, y_batch in train_dataset:

Attention Mechanism — Q, K, V
Scaled Dot-Product Attention
Attention(Q,K,V) = softmax(QKᵀ / √dₖ) · V
Q
Query
"What am I looking for?" — current token's representation being projected.
K
Key
"What do I have?" — all tokens' keys. QKᵀ → similarity scores.
V
Value
"What do I actually give?" — weighted by softmax(QKᵀ/√dₖ).
Positional Encoding
Sinusoidal (original)
PE(pos,2i) = sin(pos / 10000^(2i/d_model))
PE(pos,2i+1) = cos(pos / 10000^(2i/d_model))
💡
Learned Positional Embeddings (used in BERT, GPT): Just another Embedding layer! tf.Variable(shape=(max_seq_len, d_model)) — trainable positions. Often performs better than sinusoidal.
🔥
Masking in Transformers: Causal mask (GPT): upper triangular matrix of -inf to prevent attending to future tokens. Padding mask: -inf for pad tokens to ignore them in attention.
# ── Full Transformer Encoder Block ───────────
class EncoderBlock(keras.layers.Layer):
    def __init__(self, d_model=512, num_heads=8, dff=2048, dropout=0.1):
        super().__init__()
        self.mha   = keras.layers.MultiHeadAttention(num_heads=num_heads, key_dim=d_model//num_heads, dropout=dropout)
        self.ffn1  = keras.layers.Dense(dff, activation="gelu")
        self.ffn2  = keras.layers.Dense(d_model)
        self.norm1 = keras.layers.LayerNormalization(epsilon=1e-6)
        self.norm2 = keras.layers.LayerNormalization(epsilon=1e-6)
        self.drop1 = keras.layers.Dropout(dropout)
        self.drop2 = keras.layers.Dropout(dropout)

    def call(self, x, mask=None, training=False):
        # 1. Multi-Head Self-Attention
        attn = self.mha(x, x, attention_mask=mask, training=training)
        x = self.norm1(x + self.drop1(attn, training=training))   # Add & Norm
        # 2. Feed-Forward Network
        ffn = self.ffn2(self.ffn1(x))
        x = self.norm2(x + self.drop2(ffn, training=training))    # Add & Norm
        return x

# ── Build Causal Mask (for GPT-style decoder) ─
def causal_mask(seq_len):
    mask = 1 - tf.linalg.band_part(tf.ones((seq_len, seq_len)), -1, 0)
    return mask * -1e9   # large negative → 0 after softmax

Normalization stabilizes training by keeping activations in a healthy range. Regularization prevents overfitting by penalizing complexity.
BatchNormalization
Formula
x̂ = (x − μ_batch) / √(σ²_batch + ε)
y = γ·x̂ + β

⚠️

BN vs LayerNorm vs GroupNorm
BatchNorm
  • Normalizes across batch dimension
  • Great for CNNs with large batches
  • Bad for small batches (<16) or RNNs
  • Different behavior train vs inference
LayerNorm
  • Normalizes across feature dimension
  • Same behavior train and inference
  • Standard in Transformers/NLP
  • Works with any batch size, even 1
GroupNorm
  • Divides channels into groups, normalizes each
  • Good middle ground for small batches
  • Used in detection models (Mask R-CNN)

class F1Score(keras.metrics.Metric):
    def __init__(self, name="f1", **kwargs):
        super().__init__(name=name, **kwargs)
        # add_weight: creates a Variable tracked by metric
        self.tp = self.add_weight(name="tp", initializer="zeros")
        self.fp = self.add_weight(name="fp", initializer="zeros")
        self.fn = self.add_weight(name="fn", initializer="zeros")

    def update_state(self, y_true, y_pred, sample_weight=None):
        # Called every batch — accumulate counts
        y_pred = tf.cast(y_pred > 0.5, tf.float32)
        y_true = tf.cast(y_true, tf.float32)
        self.tp.assign_add(tf.reduce_sum(y_true * y_pred))
        self.fp.assign_add(tf.reduce_sum((1-y_true)*y_pred))
        self.fn.assign_add(tf.reduce_sum(y_true*(1-y_pred)))

    def result(self):
        # Called at end of epoch — compute final value
        precision = self.tp / (self.tp + self.fp + 1e-7)
        recall    = self.tp / (self.tp + self.fn + 1e-7)
        return 2 * precision * recall / (precision + recall + 1e-7)

    def reset_state(self):
        # Called at start of each epoch — reset accumulators
        self.tp.assign(0); self.fp.assign(0); self.fn.assign(0)

LSTM Gates — Internals
LSTM equations
f_t = σ(W_f·[h_{t-1}, x_t] + b_f) ← forget gate
i_t = σ(W_i·[h_{t-1}, x_t] + b_i) ← input gate
C̃_t = tanh(W_C·[h_{t-1}, x_t] + b_C) ← candidate
C_t = f_t·C_{t-1} + i_t·C̃_t ← cell state
o_t = σ(W_o·[h_{t-1}, x_t] + b_o) ← output gate
h_t = o_t · tanh(C_t) ← hidden state

CONCEPT 33 + 34
Transfer Learning & Fine-Tuning
Use a model pretrained on millions of images as a feature extractor, then fine-tune it on your smaller dataset. The most practical way to get high accuracy with limited data.
The 3-Phase Transfer Learning Recipe
01
Load pretrained base — freeze everything
Load ResNet/EfficientNet/etc with include_top=False. Set base_model.trainable = False. Only your new head trains. Fast convergence, no catastrophic forgetting.
02
Train the head — high learning rate
Compile with lr=1e-3, fit for 10–20 epochs. The head learns to interpret frozen features for your task.
03
Unfreeze top layers — very small LR
Set base_model.trainable = True, recompile with lr=1e-5 (10-100× smaller). Fine-tune the top N layers. Lower layers keep general features.
Pretrained Model Parameters
ParameterOptionsDescription
weights"imagenet" / None / path"imagenet" = pretrained on 1M images. None = random init. Path = custom weights file.
include_topTrue / FalseFalse = remove final classification layer. Always False for transfer learning to your task.
input_shape(H, W, 3)Override default input size. EfficientNetB0 default: 224×224. Set to your image size.
poolingNone/"avg"/"max"Global pooling after last conv block. "avg" → (batch, features). None → keep spatial dims.
classes1000Only used when include_top=True. Set to your number of classes for fine-tuning the top.
classifier_activation"softmax"Final activation when include_top=True. Set None if using from_logits=True in loss.

lr_schedule = keras.optimizers.schedules.CosineDecay(
    initial_learning_rate=0.0,   # start from 0
    decay_steps=total_steps,
    warmup_target=1e-3,           # peak LR
    warmup_steps=warmup_steps      # linear increase phase
)
optimizer = keras.optimizers.Adam(learning_rate=lr_schedule)
CONCEPT 29 + 30
Mixed Precision & Multi-GPU
Mixed Precision Training
Use float16 for forward/backward pass (faster, less memory), but keep float32 for weight updates (precision). 2–3× speedup on modern GPUs with Tensor Cores.
1
Set global policy
keras.mixed_precision.set_global_policy("mixed_float16") — affects all new layers.
2
Use float32 output layer
Final Dense must use float32: Dense(10, activation="softmax", dtype="float32"). Prevents underflow in softmax.
3
Wrap optimizer
LossScaleOptimizer(optimizer) scales loss up to prevent float16 gradient underflow, then scales back.
from tensorflow.keras import mixed_precision

# Set globally — all layers now use float16 compute
mixed_precision.set_global_policy("mixed_float16")

model = keras.Sequential([
    keras.layers.Dense(512, activation="relu"),  # float16
    keras.layers.Dense(10, activation="softmax", dtype="float32")  # float32!
])

optimizer = keras.optimizers.Adam(1e-3)
optimizer = mixed_precision.LossScaleOptimizer(optimizer)
Multi-GPU: MirroredStrategy
Synchronous data parallelism. Each GPU gets a copy of the model and a different batch slice. Gradients are aggregated (AllReduce) and applied identically to all GPUs.
StrategyUse Case
MirroredStrategyMultiple GPUs on single machine. Most common. Automatic AllReduce.
MultiWorkerMirroredStrategyMultiple machines, each with GPUs. For very large distributed training.
TPUStrategyGoogle Cloud TPUs. Requires TPU runtime.
ParameterServerStrategyAsync training with parameter servers. For heterogeneous clusters.
# ── MirroredStrategy — single machine ────────
strategy = tf.distribute.MirroredStrategy()
print(f"Devices: {strategy.num_replicas_in_sync}")

with strategy.scope():        # ALL model/optimizer creation here!
    model = build_model()
    model.compile(
        optimizer=Adam(1e-3),
        loss="sparse_categorical_crossentropy"
    )

# Batch size scales with number of GPUs!
BATCH_PER_GPU = 64
GLOBAL_BATCH = BATCH_PER_GPU * strategy.num_replicas_in_sync
model.fit(train_ds.batch(GLOBAL_BATCH), epochs=50)

Data Augmentation
Artificially expand your training set by applying random transformations. Images that look different but contain the same information. Critical to prevent overfitting on small datasets.
Keras Augmentation Layers — Parameters
LayerKey ParametersDescription
RandomFlipmode="horizontal"/"vertical"/"horizontal_and_vertical"Randomly flip images. "horizontal" is almost always safe. Vertical only for satellite/medical.
RandomRotationfactor=0.1 (fraction of 2π), fill_mode="reflect"/"wrap"/"constant"Rotate by random angle up to factor×360°. factor=0.1 = ±36°.
RandomZoomheight_factor=0.1, width_factor=NoneRandomly zoom in or out. Negative values = zoom in, positive = zoom out.
RandomTranslationheight_factor, width_factor, fill_modeShift image up/down/left/right randomly. Helps with off-center objects.
RandomContrastfactor=0.2Randomly adjust contrast: contrast_factor ∈ [1-f, 1+f].
RandomBrightnessfactor=0.2, value_range=(0,1)Add random brightness offset. value_range should match your image normalization.
RandomCropheight, widthCrop to fixed size at random location. Good for images larger than model input.
Rescalingscale=1/255., offset=0.0Normalize pixel values. scale=1/255 maps [0,255]→[0,1]. offset=-1 with scale=1/127.5 → [-1,1].
RandomCutoutheight_factor, width_factorZero out random rectangular patches. Strong regularization.
MixUp / CutMixvia custom layerAdvanced: blend two images and their labels. State-of-the-art for ImageNet training.
# ── Augmentation as part of model (best practice) ─────
data_augmentation = keras.Sequential([
    keras.layers.RandomFlip("horizontal"),
    keras.layers.RandomRotation(0.05),
    keras.layers.RandomZoom(0.1),
    keras.layers.RandomContrast(0.1),
], name="augmentation")

# These layers are ONLY active during training=True
inputs = keras.Input((224, 224, 3))
x = data_augmentation(inputs, training=True)  # augment on GPU
x = keras.layers.Rescaling(1./255)(x)
outputs = base_model(x)

# Alternative: augment in tf.data pipeline (CPU)
train_ds = train_ds.map(
    lambda x, y: (data_augmentation(x, training=True), y),
    num_parallel_calls=tf.data.AUTOTUNE
)
